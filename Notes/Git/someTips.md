# Git-commit 提交规范

> ``bulid`` : 影响构建系统或外部依赖项的更改 (example scopes: gulp, broccoli, npm)

> ``ci`` : 对我们的CI配置文件和脚本的更改 (example scopes: Circle, BrowserStack, SauceLabs)

> ``docs`` : 仅文档更改

> ``feat`` : 一项新功能

> ``fix`` : 一个错误修复

> ``perf`` : 改进性能的代码更改

> ``refactor`` : 既不修正错误也不增加功能的代码更改

> ``test`` : 添加缺失的测试或更正现有的测试
